# Agent Instructions

## Mandatory project logging
For every implementation interaction, append exactly one new entry to `docs/context/cumulative-context.md`.

## Required entry fields
- date
- interaction_id
- tags
- what
- how
- why
- files
- next

## Coding quality policy
- Keep comments short and clear.
- Document each function, method, and background process behavior.
- Prefer native lightweight solutions and avoid unnecessary dependencies.

## Scope policy
- Preserve binary-first architecture for Raspberry Pi.
- Avoid adding web runtimes unless explicitly approved.
